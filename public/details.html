1. システム概要

1.1 アーキテクチャ

本システムは以下の技術スタックで構成されています：

  • フロントエンド: React 19 + Vite
  • 顔検出: MediaPipe Face Mesh（468ランドマーク）
  • チャート描画: Chart.js（Radar）
  • 音声: Web Audio API

1.2 処理フロー

  1. キャリブレーション: 正面顔の基準データを取得（2秒静止）
  2. 回旋測定: 右→左→上の3方向の首回旋を測定
  3. 特徴抽出: 各フレームから顔・首・肩の特徴を抽出
  4. スコア算出: 複数の指標を統合して戦闘力を算出
  5. 適性推論: ファジィ論理で6競技への適性を推論

2. 顔検出・ランドマーク抽出

2.1 MediaPipe Face Mesh

Google MediaPipe の Face Mesh モデルを使用し、顔から468個の3Dランドマークを検出します。

2.2 角度計算

顔の回旋角度（yaw, pitch）は、目の位置と鼻の位置から計算します。

Yaw（左右回旋）:
yaw = atan2(rightEye.z - leftEye.z, rightEye.x - leftEye.x) × (180/π)
Pitch（上下回旋）:
pitch = atan2(nose.y - eyeCenter.y, nose.z - eyeCenter.z) × (180/π) - 90

3. スムーズネス計算

3.1 概要

首回旋の滑らかさを数値化します。ガクつきや微停止が少ないほど高スコアになります。

3.2 評価区間

最大回旋角度の10%〜90%の区間のみを評価対象とします。開始・終了付近のノイズを除外するためです。

3.3 ガクつき検出（Jerk Detection）

連続する3フレームから角速度を計算し、以下の条件でガクつきを検出します：

  • 符号反転: 角速度の符号が反転し、かつ両方の絶対値が5°/s以上
  • 加速度急変: 角加速度の絶対値が100°/s²以上
角速度 v = (angle[i] - angle[i-1]) / dt
角加速度 a = (v2 - v1) / ((dt1 + dt2) / 2)

3.4 微停止検出（Micro-Stop Detection）

一時的に速度が落ちて再び上がるパターンを検出します：

  • 条件: v1 < 0.01 かつ v2 > 0.05

3.5 スコア計算

jerkPenalty = min(jerks × 5, 40)
stopPenalty = min(microStops × 3, 20)
score = max(0, 100 - jerkPenalty - stopPenalty)

4. ネック指数

4.1 静的ネック指数

正面顔の顎幅と顔幅の比率から首の太さを推定します。

ratio = jawWidth / faceWidth

5. 首肩ボリューム特徴

5.1 概要

首から肩にかけてのボリューム感を3つの指標で評価します。

5.2 肩幅（Shoulder Width）

耳の位置と顎の位置から肩幅を推定します。

estimatedShoulderWidth = earWidth × 1.3 + jawWidth × 0.2
widthRatio = estimatedShoulderWidth / faceWidth

スコアリング: 比率 1.2〜1.6 を 0〜100 に正規化

5.3 肩傾斜（Shoulder Slope）

首から肩へのラインの傾斜角度を測定します。傾斜が緩やかなほど高スコア。

5.4 肩面積（Shoulder Area）

首から肩にかけての面積比を推定します。

5.5 総合スコア

score = (widthScore + slopeScore + areaScore) / 3

6. 闘争的フェイス特徴

6.1 概要

顔の形態的特徴から5つの指標を計算します。30フレームの履歴で平均化し、安定した計測を実現します。

6.2 fWHR（顔幅比）

Facial Width-to-Height Ratio。頬骨幅 / 顔の高さ（眉下〜上唇）の比率。

fWHR = cheekboneWidth / faceHeight
通常値: 1.4〜2.4
スコア: (fWHR - 1.4) / (2.4 - 1.4) × 100

6.3 表情抑制（Expression Suppression）

口の開き、眉の動きなどの表情変化が少ないほど高スコア。

6.4 視線安定（Gaze Stability）

視線のブレが少ないほど高スコア。虹彩位置の標準偏差から計算。

6.5 顎ライン（Jaw Line）

顎のラインの角度とシャープさを評価。

6.6 頬骨（Cheekbone）

頬骨の突出度と位置を評価。

6.7 総合スコア

totalScore = (expression + gaze + fwhr + jaw + cheekbone) / 5

7. 耳解析（内部特徴量）

7.1 概要

耳まわりの形状から特徴量を抽出し、6競技適性の計算に使用します。

この特徴量は内部計算用であり、UIには表示されません。 6競技適性（セクション8）の入力特徴量 EJ_raw, FA_crop として使用されます。

7.2 計算される指標

  • 耳輪郭の不規則さ: 輪郭線の角度変化の標準偏差
  • 左右非対称: 左右の耳の形状差
  • 局所的膨らみ: 耳の特定部位の膨らみ
  • 輪郭連続性: 輪郭線の不連続点

7.3 6競技適性への寄与

jujutsuScore = (contour + asymmetry + thickness + continuity) / 4

// 6競技適性で使用される変数
EJ_raw = jujutsuScore / 100 // 柔術・レスリング・MMAで使用
FA_crop = asymmetry.score / 100 // ボクシング・MMA・キックで使用

8. 6競技適性（ファジィ推論）

8.1 概要

6つの格闘技（ボクシング、キック/ムエタイ、MMA、レスリング、柔道、柔術）への適性を、線形スコアリングとファジィ補正の組み合わせで算出します。

クラスタリングや機械学習は使用していません。固定の重み付け線形結合＋ファジィ補正のみです。

8.2 入力特徴量（正規化 0-1）

変数内容ソース
NS_raw静的ネック指数calibration
NV_raw首肩ボリュームtrapezius.score
TT_raw体幹厚みtrapezius.area
SH_raw肩幅/頭幅比trapezius.width
EJ_raw柔術スコアjujutsu.score
FA_crop顔の非対称性jujutsu.asymmetry
NJ_crop首顎角度jawWidth/faceHeight

8.3 線形スコアリング（凍結版）

各競技への適性スコアは、特徴量の重み付き線形結合で計算します。

// 柔道（rawのみ）
score_judo = 0.45×NV_raw + 0.35×NS_raw + 0.20×SH_raw

// レスリング（rawのみ）
score_wrestling = 0.35×NV_raw + 0.30×NS_raw + 0.20×TT_raw + 0.15×EJ_raw

// 柔術（croppedのみ）
score_bjj = 0.40×EJ_crop + 0.25×NS_crop + 0.20×SH_crop + 0.15×NV_crop

// MMA（混合）
score_mma = 0.25×NV_raw + 0.20×NS_raw + 0.15×TT_raw + 0.20×FA_crop + 0.20×EJ_crop

// ボクシング（croppedのみ、マイナス項あり）
score_boxing = 0.35×FA_crop + 0.25×NJ_crop - 0.25×NV_crop - 0.15×TT_crop

// キック/ムエタイ（混合）
score_kick_muay = 0.30×NJ_crop + 0.30×NS_raw + 0.20×TT_raw + 0.20×FA_crop

8.4 ファジィメンバーシップ関数

High メンバーシップ（固定閾値版）

μ_high(x, threshold=0.6) =
  0                               (x ≤ threshold)
  (x - threshold) / (1 - threshold)   (threshold < x < 1)
  1                               (x ≥ 1)

High メンバーシップ（パーセンタイル版・柔道専用）

μ_high_pct(x, p90, xMax) =
  0                         (x ≤ p90)
  (x - p90) / (xMax - p90)   (p90 < x < xMax)
  1                         (x ≥ xMax)

Medium メンバーシップ

μ_medium(x) =
  0                     (x ≤ 0.3 or x ≥ 0.7)
  (x - 0.3) / 0.2       (0.3 < x < 0.5)
  (0.7 - x) / 0.2       (0.5 ≤ x < 0.7)

8.5 ファジィ補正ルール

MAX_FUZZY_BONUS = 10  // 最大ボーナス点

// ① 柔道
IF NV_raw is High(pct) AND NS_raw is High(pct)
THEN bonus_judo = 10 × min(μ_nv, μ_ns)

// ② レスリング
IF NV is High AND NS is High: base = 6 × min(μ_nv, μ_ns)
IF EJ is Medium or High: ear_bonus = 4 × max(μ_medium, μ_high)
bonus_wrestling = min(10, base + ear_bonus)

// ③ 柔術
IF EJ_crop is High
THEN bonus_bjj = 10 × μ_high(EJ_crop)

// ボクシング、MMA、キック/ムエタイはファジィ補正なし

8.6 最終スコア

finalScore[sport] = linearScore[sport] + fuzzyBonus[sport]
(0〜100にクリップ)

8.7 ボクシングの正規化

ボクシングは負の項を含むため、特別な正規化を適用：

normalized = (rawScore + 0.30) / 0.90

9. 戦闘力計算

9.1 ベーススコア

6つの内部スコアの平均値をベーススコアとします。

baseScore = (rightSmoothness + leftSmoothness + upSmoothness
            + staticNeckIndex + dynamicNeckIndex + shoulderVolume) / 6

9.2 指数関数スケーリング

ベーススコアを指数関数でスケーリングし、少年漫画風の大きな数値で戦闘力を算出します。

combatPower = A × exp(B × baseScore)

A = 1
B = 0.13

スケーリング例

ベーススコア戦闘力
10約 4
30約 50
50約 665
70約 8,800
90約 116,000
100約 4億

10. ガンマ補正

10.1 目的

表示用スコアの分布を調整するため、ガンマ補正を適用します。内部スコアには影響しません。

10.2 静的ネック指数のガンマ補正

γ = 1.2
displayScore = (rawScore / 100)^γ × 100

γ > 1 によりスコアを抑制し、高スコアを出にくくします。

10.3 左右バランスのガンマ補正

γ = 1.3
displayScore = (rawScore / 100)^γ × 100
